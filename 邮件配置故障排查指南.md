# 邮箱服务配置切换指南

## 当前问题
`java.net.SocketException: Connection reset` 通常是由于邮件服务器连接问题导致。

## 解决方案汇总

### 方案A：修复QQ邮箱配置（推荐优先尝试）

1. **重新生成授权码**
   - 登录 https://mail.qq.com
   - 设置 → 账户 → POP3/IMAP/SMTP服务
   - 关闭后重新开启，获取新授权码
   - 更新 application.yaml 中的 password 字段

2. **已优化的配置**
   - 端口：465（SSL加密）
   - 协议：smtps
   - 超时：30秒
   - 调试：开启

### 方案B：切换到163邮箱

#### 1. 注册/登录163邮箱
访问：https://mail.163.com

#### 2. 开启SMTP服务
- 设置 → POP3/SMTP/IMAP
- 开启"IMAP/SMTP服务"
- 设置客户端授权密码

#### 3. 修改 application.yaml
```yaml
spring:
  mail:
    host: smtp.163.com
    port: 465
    username: your-email@163.com
    password: your-authorization-code  # 163邮箱的授权码
    protocol: smtps
    default-encoding: UTF-8
    connectiontimeout: 30000
    writetimeout: 30000
    timeout: 30000
    properties:
      mail:
        smtp:
          auth: true
          ssl:
            enable: true
            trust: smtp.163.com
        debug: true
```

### 方案C：切换到Gmail

#### 1. 开启两步验证
- 访问 Google Account → Security
- 开启"两步验证"

#### 2. 生成应用专用密码
- 访问：https://myaccount.google.com/apppasswords
- 选择"邮件"和设备
- 生成16位应用密码

#### 3. 修改 application.yaml
```yaml
spring:
  mail:
    host: smtp.gmail.com
    port: 587
    username: your-email@gmail.com
    password: xxxx-xxxx-xxxx-xxxx  # 16位应用密码
    protocol: smtp
    default-encoding: UTF-8
    connectiontimeout: 30000
    writetimeout: 30000
    timeout: 30000
    properties:
      mail:
        smtp:
          auth: true
          starttls:
            enable: true
            required: true
          ssl:
            enable: false
        debug: true
```

### 方案D：临时禁用邮件功能

如果只是测试其他功能，可以暂时注释掉邮件相关代码：

在 EmailServiceImpl.java 中：
```java
@Override
public boolean sendVerifyCode(String toEmail, String verifyCode) {
    log.warn("邮件功能已禁用 - 验证码: {} (目标: {})", verifyCode, toEmail);
    return true; // 直接返回成功
}
```

## 常见错误代码及解决

| 错误代码 | 含义 | 解决方法 |
|---------|------|---------|
| 535 | 认证失败 | 检查授权码是否正确 |
| 550 | 邮箱不存在 | 检查发件人邮箱地址 |
| 553 | 邮箱地址格式错误 | 检查邮箱格式 |
| 421 | 服务不可用 | 稍后重试或切换邮箱 |
| Connection reset | 连接被重置 | 检查网络和防火墙 |
| Connect timed out | 连接超时 | 检查网络和端口 |

## 推荐的开发环境配置

对于开发环境，建议：
1. 使用163邮箱（相对稳定，授权码容易获取）
2. 开启debug模式查看详细日志
3. 设置合理的超时时间（30秒）
4. 添加详细的异常处理和日志记录

## 生产环境建议

1. 使用企业邮箱服务（如腾讯企业邮、阿里企业邮）
2. 配置邮件发送队列，避免阻塞主线程
3. 添加重试机制
4. 监控邮件发送成功率
5. 关闭debug模式
